Getting things done: Programming

By Tino Tyson on June 1, 2025

Learning to program is a tedious task on its own. One has to learn to walk from sratch. Its like reinventing the wheel for some, whilst it may feel like using a passed over tool for others. Many see programming as an endless loop of tutorials, and most believe its all about the languages.

Since there are a lot to select from and to put forward, most of the starters often tend to search up something like "Best programming language to get a high paying job in 2025", "What is the current trend in programming?", "Is coding necessary for a software engineer?"

Many others align with the pessimistic side of things and belive that programming is not their cup of tea, or that the jobs in the software world is finished since the rise of AI.

These fears and doubts are correct and necessary only up to a certain level.

Why do we fall?

It comes as no surprise that everyone should be learning atleast the basics of programming in today's world.

Most people rather not want to learn. This can be due to various reasons such as lack of trust to this side of the economic world, talks on the lack of employment found in this field, procastination, fear of tutorial hell or even the fact that learning this skill is monetarily very expensive.

So how do we learn to program?

The easiest and most humanly possible way to create it is by getting things done. You cannot become a painter just by painting a single image. You can't become a chef just by making one single dish. Similarly in programming, you can't become an actual programmer, without doing some projects. You don't have to look or know what you are doing or how you are doing is what that matters.

Did you make a mistake? Good! Write some more code.
Did it not reach the level you were expecting? Good! Make it even better.
Is the code you are writing, not working properly? Good! Try to rewrite the compiler and destroy the OS in the process.

Project based learning 101

Basic steps for projects:

  • Portfolio: An absolute must to your collection of projects. The first project, you would have to make is a portfolio, because it is the introduction, to you personal brand. A portfolio also allows others to see what you have made or done.
  • Web App: Try to make a functioning web app. Interesting on a small scale as a project, but powerful enough to run as a fully functioning business model, on its own. Many web apps often run as large scale SaaS.
  • Mobile App: Similar to Web App, but on smartphones. Can be both developed using tools, as well as from scratch.
  • Participating in a CTF: CTFs build a strong foundation in the field of cybersecurity and hacking. One can learn a lot from these events.
  • Trying things on the OS: Great way to find out how an Operating System works. Even better way to destroy it completely. Better done using a virtual machine like Virtualbox or VMware, for obvious reasons.

Example: Development of a Portfolio

The following steps need not be strictly followed:

  • HTML: For creating the basic structure of the website.
  • CSS: To implement the necessary designs, that align with the occupation you do. If you mainly focus on fronte end, then you can make it more appealing to the human eye. Else a simple, yet clean design would suffice the criteria.
  • JavaScript: It is just as important to implement necessary functions, depending on how you wish your website should be. Go on! Implement a dark mode, a newsletter box, or any other feature. It is necessary, especially if you work in similar fields to provide User Experience(UX).
  • Others: Add multiple other features, in order to make a more interesting and useful website.

Will this be enough?

While it may be hard to begin and continue your journey, it is a hard truth you should face in order to achieve something. When I say a hard truth I don't mean that you should be doing projects to learn, but it is a bonus on your part. Doing projects would allow you to shine more in an interview or in the process of career selection. Also it would help in the creating a strong foundation of the basic and advanced topics.
If you personally ask me if this would be enough, then I don't know. No one knows how a person's life would turn out. The main thing I want you to take from this is that, you should trust the process. No matter how boring or hard it gets, you will see the results. But consistency is key; or not I don't know. Engineers, especially software engineers are supposed to be lazy enough to create solutions others are not thinking, whilst still being deligent enough to finish what they have started.

Go Touch Grass!

The most important thing to do as a software engineer is to go outside and touch grass. Always on the screen will get you nowhere. Priority no.1 is to exist in the real world. If you are unable to enjoy the real world, then none of this is worth anything. Most of us are in this for the money. If you can't even take a break and slow down for a some time, then all this is for nothing. Even in the future, you won't be able to enjoy the fruits of your labour.
So always remember to take a break once in a while and go outside. Meet new people. Enjoy being a human. Enjoy nature. Have other hobbies other than this, because it is the real world is much more than a screen. You can learn a lot of things online, and try to implement them in real world problems, which you find outside the screen (or even online. You get the point right?)

I don't think I would have to write down an entire conclusion here. Creating this blog in itself is hard, now I'm not gonna write down a conclusion. I hope that the reader must have understood what I'm trying to convey here.
Now go! Read another post or something.
